liunx常见命令：

cd:改变目录

cd..:返回上一级

pwd:显示当前所在目录路径

ls:显示当前目录所有文件

chmod:使用 “+” 设置权限，使用 “-“ 用于取消，“=”用于给定一个权限。其中r是读权限，w是写权限，x是执行权限。ugo是代表权限的范围，u是文件或目录的所有者，g是用户组，o是其他用户，a是所有用户

rm:删除一个文件

mkdir:删除一个文件夹

mv:移动文件

grep+关键词+文件路径:根据关键词查询文件中的内容

top:查看cpu和内存

kill:终止某个进程

git：

git init:初始化仓库

git add:添加文件到暂存区

git commit:将暂存区内容添加到仓库中

git pull:下载远程代码并合并

git push:上传远程代码并合并

git checkout:切换分支

git merge:合并分支后git提交日志会显示每个分支的提交记录，显得比较乱

git rebase:合并分支后会将每个分支的提交记录合并，这时git提交记录会呈一条直线，比较干净

git clone:将远程服务器上的仓库完全镜像一份到本地

git工作区域：

workspace：工作区，平时存放代码的地方

index/stage：暂存去，临时存放改动

repository：本地仓库，存放数据的位置，存放着提交到所有版本的数据，其中head指向最新版本

remote：远程仓库，托管代码的服务器

docker命令：

docker ps 查看正在运行的容器

docker ps -a 查看所有已经创建的容器

docker images 列出镜像列表

docker start 启动容器

docker run 创建一个新的容器并运行一个命令

docker restart 重启容器

docker exec 在运行的容器中执行命令

docker stop 停止容器

docker rm 删除容器

docker rmi 删除镜像

nginx作用：

1.反向代理服务器，将请求转发到后端服务器

2.负载均衡，通过负载均衡算法，比如说轮询，加权轮询，最小连接数等来转发请求

nginx部署前后端分离项目：

将前端项目打包，放到liunx服务器一个指定的目录下，在nginx配置文件中的location下面中的root改成存放前端项目的地址。将后端项目打成jar包，使用java -jar在服务器上运行，修改nginx配置文件将代理地址改成后端项目运行的服务器ip和端口号

jekins部署后端项目流程：

  1. 下载并安装jdk，maven，git，jenkins
  2. 在jenkins客户端配置好jdk，maven和git
  3. 下载自动化部署相关插件
  4. 新建一个item打包项目。需添加git仓库地址和分支
  5. 发布。配置好jar包的地址和需要部署服务器的地址还有执行命令或脚本

docker部署springcloud项目：

  1. 创建Dockerfile文件，在项目根目录下创建一个 Dockerfile 文件，定义如何构建 Docker 镜像
  2. 将项目打成jar包
  3. 使用 Dockerfile 构建 Docker 镜像
  4. 运行docker容器
  5. 如果Spring Cloud 项目需要与其他服务交互（如 Eureka、Config Server），可以在 docker-compose.yml 中定义多个服务的配置，简化管理
