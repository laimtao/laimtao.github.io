Vue生命周期

  1. ‌创建阶段‌：
  • beforeCreate：实例初始化之后，还未开始创建DOM和挂载，此时data和methods还未初始化。
  • created：实例创建完成，已经可以使用data中的数据，但DOM还未挂载。
  • beforeMount：在挂载之前，通常在这个阶段进行一些数据的预处理。
  • mounted：组件已经挂载到DOM上，可以进行DOM操作。
  1. ‌更新阶段‌：
  • beforeUpdate：数据更新之前调用，可以在这个阶段进行一些预测或准备操作。
  • updated：数据和DOM已经更新完毕。
  1. ‌销毁阶段：
  • beforeDestroy：实例销毁前调用，可以进行一些清理工作。
  • destroyed：实例已经销毁。

var let const 的区别

  • var：允许重新赋值和重新声明。
  • let：允许重新赋值，但不允许重新声明同一作用域内的变量。
  • const：不允许重新赋值或重新声明，变量必须在声明时初始化

v-show和v-if区别

v-if:

  • 根据条件动态地渲染或销毁 DOM 元素
  • 因为涉及到 DOM 元素的创建和销毁，所以 v-if 的性能开销相对较大，适用于条件变化不频繁的情况

v-show:

  • 根据条件动态地切换 DOM 元素的显示或隐藏状态，即使条件为 false，相关的 DOM 元素仍然会保留在 DOM 树中，只是通过设置 CSS 的 display 属性来控制元素的显示和隐藏
  • v-show 的性能开销较小，适合需要频繁切换显示状态的场景
